JavaScript中的作用域与闭包

作用域

词法作用域

JavaScript使用的是词法作用域。每个代码块，函数，乃至整个脚本都有一个“词法环境”，记录了当前作用域内的局部变量等内容，以及对外部词法环境的引用。

添加代码后，情况就稍有不同。

在每个函数运行时，会创建一个新的词法环境来追踪一系列局部变量和参数。

首先将函数声明 greet 添加到全局词法环境中，现在全局词法环境里记录有 saying 变量和 greet 函数。

之后调用这个函数，会创建一个新的词法环境，也就是说现在有两个词法环境：全局词法环境和函数 greet 的词法环境。

可以看到将一个字符串作为参数传给了 greet 函数，现在 greet 函数的词法环境里有了传入的 name ，因此可直接访问。但访问函数内部没有的变量 saying ，JavaScript引擎就会向外搜索，在外部词法环境里找到了 saying 变量。这时 greet 函数便有了对外界词法环境的引用。最终结合变量，输出结果。

作用域链

如果在当前作用域里没法找到某个变量，JavaScript引擎就会在外层嵌套的作用域继续查找，直到找到该变量，或者到达最外层作用域（全局作用域）为止。

最内层的 swapNum 作用域内最直接可以访问到的的变量是 temp （毕竟就在这个作用域里进行的定义），如果想访问 num1 或者 num2 ，发现当前作用域内没有，就会向外查找，最后发现在 changeNum 内定义了 num2 ，在最外层的全局作用域里定义了 num1 。

同理，在 changeNum 作用域中可以直接访问 num2 ，想要访问 num1 时需要向上查找。但是此处无法访问到 temp。而最外层的全局作用域里只能访问 num1 ，是无法访问到 num2 和 temp 的。

在脑海中可以抽象为一根链条连接着各个作用域，内部作用域查找自己作用域内没有的变量时即为顺着这根链条向上查找。

闭包

有了上面对于词法作用域和作用域链的基础，我们可以开始了解闭包了。

首先闭包的定义是：一个函数和其关联的词法环境的组合叫做闭包。

闭包可以让函数内部访问到外部环境的变量，在计数器这种需求下挺有帮助。

结合之前的例子可以看得出来闭包的原理是作用域链。我们就用计数器这个例子来捋捋闭包。

在每个 makeCounter 函数执行后，创建了新的词法环境，其中记录了 num 变量的情况。然后它返回了一个只有一行的函数 return num++ ，存储在 counter 变量中。

当执行 counter 函数时，函数内引用了 num 变量，但是函数内部作用域并没有这个变量，于是它向外查找，发现它外部的词法环境中记录了 num 变量的情况，于是在这个词法环境里更新变量（自增1），之后的多次调用也是重复上述过程。

至于 anotherCounter ，则是很好的说明了每个函数的执行后都会创造一个新的词法环境，它们之间互不影响，因此可以分开计数。

在这个案例中 counter 和 anotherCounter 两个函数与其对应的词法环境（内部记录有 num 变量）组成了闭包。
