Vue基础学习笔记

年轻人的第一款JavaScript框架（

写在最前

多读读官方文档，Vue提供的文档真的质量挺高。此处记录的是自己学习过程中的思考与笔记。

Vue特点

  • MVVM架构：专注于数据驱动视图的方式来构建用户界面。
  • 双向数据绑定：数据和界面同时变化，不论修改哪个另外一边都能接收到变化。
  • 渐进式：从少到多，你可以一开始只用一部分Vue的功能，后期再慢慢添加。
  • 自底向上逐层应用：可以和其他的类库，框架等结合使用，而且这些工具的加入对其本身影响不会太大。
  • 声明式渲染：和HTML类似的模板语法，你只管写你想要的结果，过程交给Vue处理。
  • 虚拟DOM：在把数据实际渲染到页面之前会有一个内存中的虚拟DOM。先计算好是个什么情况，再根据一套内部的diff算法决定实际渲染时如何做最高效。

指令

带有 v- 开头的特殊HTML属性，可以设置标签属性，设置标签内容，绑定事件等。

指令的语法细节可以直接读 Vue2的官方API文档 ，肯定是比我自己写要来的更详细啦。

内容渲染

  • {{}}（插值表达式）
  • v-text
  • v-html

属性绑定

  • v-bind（简写为:）

事件绑定

  • v-on（简写为@）

v-on可以与特殊变量 $event 结合使用，来代替原生事件的 event 参数对象。

可以搭配事件修饰符来控制事件的触发。比如：

.prevent 阻止默认行为

.stop 阻止事件冒泡

.once 只触发一次绑定的事件

.self 只在 event.target 是元素自身时才触发事件处理函数

双向绑定

  • v-model

可以搭配修饰符对用户输入内容进行处理。比如：

.number 转换为数值类型

.trim 去除首尾的空白字符

.lazy 在 change 事件触发而非 input 事件触发时更新

条件渲染

  • v-if
  • v-show
  • v-else（需要搭配v-if使用，否则不生效）
  • v-else-if（需要搭配v-if使用，否则不生效）

v-if会动态的创建和移除DOM元素，v-show则是改变DOM元素的display属性来实现元素的显示与隐藏。

v-if有更高的切换开销，v-show有更高的初始渲染开销。

如果显示情况变化较少，用v-if。

如果需要频繁切换显示情况，用v-show。

列表渲染

  • v-for

基于一个数组来渲染一个列表结构。它使用类似 (item, index) in items 的语法来确定循环所需的单个项目，索引和数组。其中 index 参数可省略， item index items 三个参数的名字都可修改。

Vue内部会尽可能复用已存在的DOM元素来提升性能，为了让它方便追踪哪些DOM是可以复用的，可以为每一个列表项提供一个唯一的key属性。

key属性只能为字符串或数字类型，必须具有唯一性，不建议将index作为key的值（想想当你unshift一个数据进数组的时候，索引和数据的对应会发生变化），可以试试使用UUID或者NanoID作为key。

过滤器

过滤器（ filter ）常用于文本的格式化，可以在插值表达式或者v-bind中使用。

添加在js表达式的尾部，使用管道符（ | ）进行调用。

方法，计算属性与侦听器

它们都可以对数据进行处理。

方法（ methods ）和计算属性（ computed ）常用于实现对于数据的同步处理。前者是只要调用一次，不论依赖的数据是否变化都会执行并取得结果，而后者会针对于它依赖的数据进行缓存，如果这些数据没变化的话，Vue就会立刻返回之前缓存过的数据，节省开销。

而侦听器（ watcher ）是用于对于数据变化进行一些额外的事件处理，也能异步处理这些数据。

你要说啥数据都想用侦听器去处理也不是不可以，但是你会发现这样做会浪费了Vue框架给你提供的一些便利——针对数据的处理代码变得命令式（需要事无巨细的写出处理数据的每一步），所以我的总结是：

  • 确保总是拿到最新的数据，无视缓存，数据处理的开销不大，可以使用方法。
  • 因为数据处理开销较大，避免不需要的计算，需要之前缓存的数据，可以使用计算属性。
  • 数据变动进行异步操作或者额外的处理（比如定时器延时处理，或者显示提示），可以使用侦听器。

Vue实例的生命周期

官方文档中的生命周期图示

beforeCreate

this指向创建的实例。无法访问data、computed、watch、methods。

created

实例创建完成，但还未挂载到页面上。可以访问data、computed、watch、methods。不能访问$el属性。

最佳实践：声明不需要被Vue追踪的非响应式变量。

beforeMount

页面上显示的是未经过Vue处理的DOM结构，此时对DOM的操作最终都不会生效。

mounted

页面上显示的是经过Vue处理的DOM结构，此处初始化工作结束。

最佳实践：发出网络请求，开启定时器，绑定自定义事件等。

beforeUpdate

数据已经更新，页面还未更新。

updated

数据页面都已更新。

最佳实践：避免在这个钩子函数中操作数据，很容易再次触发beforeUpdate钩子函数从而陷入死循环。

beforeDestroy

实例销毁前，此时实例还是可用状态。

最佳实践：销毁定时器，解绑事件等。

destroyed

完全销毁实例，解除实例与其他实例的连接，解绑所有的指令。

非单文件组件

为什么要使用组件

不断细化一个页面中的组成部分，使其编写/维护更加容易，增加了组件的复用性，使其不容易和外界其他部分发生冲突。

就以我个人的经历来讲，有一个学期的期末考核是一个较小的纯前端项目，在开发这个项目的过程中，发现不管是首页还是子页面都有共通的元素（导航栏，页脚，菜单，部分展示区域等等）。

我尝试过将共用的元素的CSS样式文件拆分成单独的文件，再通过复制对应区域的HTML代码到不同文件里来实现“复用”。虽然在一定程度上使我的开发过程变得轻松，但是依旧存在需要到多处文件中修改同一效果的情况，很容易错改或者漏改。

这尚且是小项目+一定的思考过后产生的不成熟解决方案，如果在大型项目中，错综复杂的相互引用关系想必让人难以维护。

Vue中的组件就将单个功能区域封装到一块，对外暴露一个“整体”的组件来实现开头所说的这些优势。
