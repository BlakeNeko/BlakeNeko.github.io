Git学习笔记

虽然目前日常生活中还没有大规模用到Git来管理开发，但还是以“自己够用”为出发点学习并记录了此笔记。

Git是什么

一个分布式版本控制系统。

追踪一系列文件的改动，轻松的管理它们的不同版本，满足自己回溯/保留某个最佳版本的需求。

维基百科上关于Git的介绍

在Windows上安装和初步配置Git

在这里下载Windows版本的Git，我此处直接按照默认配置一步步安装完毕。

安装完毕后在开始菜单里找到Git->Git Bash，运行后能看到一个类似命令行的窗口，在其中输入命令来配置基础信息。

基础信息配置完成后，生成公钥并添加到github里。

在你的用户主目录里找到.ssh目录，里面应该有两个文件，id_rsa是私钥，不能泄露，id_rsa.pub是公钥，可以告诉任何人。

登录github，打开Settings，打开SSH and GPG keys，点击New SSH key，Title随便取，将id_rsa.pub文件的内容粘贴在下面的Key里，再点击Add SSH key完成添加。

这几步做完以后就能方便的将代码提交到github上了。

一些概念

本地仓库由git维护的三颗“树”组成。

  1. 工作区存放实际文件。

  2. 暂存区临时保存你的改动。

  3. HEAD指向你最后一次提交的结果。

日常操作流程

使用Git

创建仓库

仓库（repository），简单理解就是一个目录，这个目录下面的文件都归Git追踪。

初始化完成后，这个目录下的.git目录是Git软件使用的，就不要随便动了。

添加/删除/重命名文件

你想追踪的文件必须要放在仓库目录里。

提交更改

管理分支

其中合并分支这一操作，大部分情况下Git都会尝试自动合并，但有时会有“合并冲突”的情况，这个时候需要你先git diff查看冲突情况，再手动去修改冲突的代码部分，git add来添加修改完后的文件，最后合并。

查看信息

在输出结果里会看到版本号 id（版本号），这个是切换版本所需的重要信息。带有HEAD的为当前版本。

撤销更改

最后一条目录，参数里的HEAD^表示上一个版本，HEAD^^则表示上上个版本，HEAD~10表示往上10个版本，以此类推。

可以使用git log或git reflog找到想切换版本的版本号。

前者用于查看版本历史记录，便于找到想回退的历史版本；后者用于查看历史提交记录，便于找到目前版本的“新”版本（万一有时候倒回老版本又后悔了呢）。
