我的hexo使用方案

记录自己搭建静态博客的步骤和日常写文章的过程。

搭建博客和写作环境

安装一系列软件

安装nodejs

没啥好说的，选择的是官网提供的LTS版本，基本保持默认配置。

安装git

同样直接安装官网提供的Windows版本git，基本保持默认配置，我选择添加了右键快捷菜单的选项。

安装Typora

一个更加直观方便的markdown编辑器，代替我之前用的vscode。

安装PicGo

目前我选择将博客所需用的图片放在github另外一个仓库里，利用这个软件来简化图片的上传过程。

搭建过程

在github上新建仓库

我选择的是github pages来托管自己的静态博客。需要在github上新建一个仓库，仓库名为你的github用户名.github.io。

配置git

运行git bash，输入以下命令来配置上传文件到博客仓库时所使用的信息。

生成公钥并添加到github里。

在你的用户主目录里找到.ssh目录，打开github的Settings，SSH and GPG keys，单击New SSH key，Title随便取，将id_rsa.pub文件的内容粘贴在下面的Key里，再点击Add SSH key完成添加。

安装hexo

先新建一个用于存放博客文件夹的目录，我选择了D:/博客，在该文件夹里鼠标右键运行git bash，之后在里面运行命令。命令的具体含义和其他帮助可以去看hexo文档。

这个时候在本地一个最基本的hexo博客就搭建完毕了，可以运行以下命令来在本地生成文件，启动服务器来预览效果。

之后的配置网站信息，更换主题，修改网站生成文章的模板，如何配置部署，常用命令什么的都可以通过阅读hexo的文档，或者你想使用主题的文档解决，在此不赘述。

配置PicGo

首先在github上新建另一个仓库，用于存放博客需要的图片。此处我选择的名字是blog-images，然后找到Settings，Developer settings，Personal access tokens，单击Generate new token，Note我写的是For PicGo，给repo单选框打勾，拉到网页最下面单击Generate token，复制接下来页面里的那串token，待会要用。

也是基本维持默认配置安装PicGo，打开后找到图床设置，展开后选择github图床，仓库名设置为BlakeNeko/blog-images，分支名设置为main，设定token填入刚刚在github上生成的那一串就行，确定，设为默认图库。

切换到PicGo设置，我开启了时间戳重命名功能，就不用自己每次命名图片文件了。

更多的配置信息同样通过看它提供的文档解决。

配置Typora

基本保持默认设置安装，运行Typora，找到菜单栏的文件，偏好设置。

通用里我设置了启动选项为打开指定目录，并添加了自己博客的_posts文件夹，这样一启动Typora就能直接定位到博文原文件目录不用自己再打开了。

图像里我设置了插入图片时...为上传图片，勾选对本地位置的图片应用上述规则。上传服务设定，上传服务设置为PicGo(app)，在PicGo路径里设置PicGo应用程序的位置就行，最后验证上传选项，成功了就说明配置成功了。

绑定域名

我是在namesilo上购买的域名。图一个便宜，免备案和支持支付宝微信付款。

首先就是正常的注册账户，把想要的域名加入购物车，填写注册域名用的实名信息。在结算界面关掉Auto-Renew（自动续费）然后给Privacy Setting选择WHOIS Privacy，结账扫码付款，域名就到手了。

在namesilo上找到Manage My Domains，选中自己的域名，单击右侧的蓝色小地球按钮管理DNS。

先删除默认的所有DNS记录，然后添加一个A记录，hostname不填，ipv4 address填写github pages的IP地址（可以用ping命令获得），TTL填3600，单击submit；之后添加一个CNAME记录，hostname填www，target hostname填写github pages的地址（github.io那个），TTL填3600，单击submit。

在博客目录的source目录下创建一个CNAME文件（无扩展名），编辑内容为你注册的域名，保存，hexo clean && hexo g && hexo d推送到仓库。

大概等个20分钟访问绑定的这个域名就能打开博客了。

使用流程

硬要说其实也没啥，掌握Typora的快捷键可以事半功倍，结合之前的工具配置能达到一个比较舒适的流程。

（不过还是没有购置VPS，搭建wordpress，typecho等博客系统，使用它那种网页端的撰写工具方便😂）

目前的博文写作流程就是先在博客文件夹下用git bash运行hexo new post 文章名，打开Typora定位到新建的文件，开始写博文。

要配图的话直接拖到Typora里再微调位置和说明就好。

写完以后hexo clean && hexo g && hexo s本地先预览下有没有问题，确认无误后再hexo d推送。
